# 🚀 部署到 Gitee Pages — 详细指南

按照本指南操作，约 **15 分钟**即可将游戏免费部署到网上，朋友用浏览器打开链接就能玩。

---

## 第一步：注册 Gitee 账号

1. 打开 https://gitee.com
2. 点击右上角「注册」
3. 用手机号或微信注册
4. **重要：** 注册后需要完成**实名认证**
   - 点击右上角头像 → 「账号设置」→ 「身份认证」
   - Gitee Pages 服务要求实名认证后才能使用

---

## 第二步：创建仓库

1. 登录后，点击右上角 `+` → 「新建仓库」
2. 填写信息：

   | 字段 | 填写内容 |
   |------|----------|
   | 仓库名称 | `english-word-rpg` |
   | 仓库介绍 | 英语单词打怪升级 RPG 游戏 |
   | 是否开源 | **公开**（必须公开才能用 Pages） |
   | 初始化仓库 | ❌ 不勾选任何选项 |

3. 点击「创建」

---

## 第三步：推送代码到 Gitee

创建仓库后会看到一个页面，显示「远程仓库地址」。类似：

```
https://gitee.com/你的用户名/english-word-rpg.git
```

复制这个地址，然后在本机命令行执行以下命令：

```bash
# 1. 进入项目目录
cd d:\vscode_projects\English_Games

# 2. 添加 Gitee 远程仓库（把下面地址换成你自己的）
git remote add origin https://gitee.com/你的用户名/english-word-rpg.git

# 3. 推送代码
git push -u origin master
```

推送时会弹出 Gitee 登录窗口，输入你的账号密码即可。

> 💡 如果不想每次输密码，可以在 Gitee 设置中添加 SSH 公钥：
> 头像 → 「设置」→ 「SSH 公钥」→ 添加你电脑的 `~/.ssh/id_rsa.pub`

---

## 第四步：开启 Gitee Pages

1. 进入你的仓库页面
2. 顶部菜单点击「服务」→ 「Gitee Pages」
3. 配置：

   | 选项 | 值 |
   |------|-----|
   | 部署分支 | `master` |
   | 部署目录 | 留空（根目录） |
   | 强制 HTTPS | ✅ 开启 |

4. 点击「启动」
5. 等待 1-2 分钟，页面会显示一个网址：

   ```
   https://你的用户名.gitee.io/english-word-rpg
   ```

6. ✅ 点击这个网址，确认游戏能正常打开！

---

## 第五步：分享链接

把这个链接发给朋友：

```
https://你的用户名.gitee.io/english-word-rpg
```

他们用手机或电脑浏览器打开就能玩！每个人的进度存在自己的浏览器里，互不影响。

---

## 📝 后续更新

当你修改了游戏代码，重新部署只需三步：

```bash
# 1. 添加改动
git add -A

# 2. 提交
git commit -m "描述你改了什么"

# 3. 推送到 Gitee
git push
```

然后进入仓库 → 「服务」→ 「Gitee Pages」→ 点击「更新」按钮。

1-2 分钟后，新版游戏就上线了！

## ⚠️ 常见问题

### Q: 打开链接显示 404？
A: 检查：
- Gitee Pages 是否已「启动」
- 仓库是否设置为「公开」
- 是否通过了实名认证

### Q: 页面能打开但是白屏？
A: 按 F12 打开开发者工具 → Console 面板，查看红色错误信息。常见原因：
- 文件没推送完整（检查仓库文件列表）
- 浏览器太旧（需支持 ES6 模块的 Chrome/Edge/Firefox）

### Q: 国内访问速度怎么样？
A: Gitee 服务器在国内，速度很快。首次加载约 1-2 秒。

### Q: 可以绑定自己的域名吗？
A: 可以。Gitee Pages 支持自定义域名（需在「服务」→「Gitee Pages」中配置）。

### Q: 存档会丢吗？
A: 存档存在用户自己的浏览器里（localStorage），与服务器无关。换浏览器/清除浏览器数据会导致存档丢失。建议定期使用「导出存档」功能备份。
